# Resume Tips for Tech Roles

## Format & Structure

- **One page** for less than 5 years of experience. Two pages max for senior roles.
- **PDF format** — it preserves formatting across systems and passes through ATS cleanly.
- **Standard section headers**: Education, Experience, Skills, Projects. ATS parsers expect these exact words.
- **Reverse chronological order** — most recent experience first.
- **No photos, graphics, or columns** — they confuse ATS parsers and waste space.
- **Consistent formatting** — same font, same bullet style, same date format throughout.

## Content

### Experience Bullets

Use the **XYZ formula**: Accomplished **X** as measured by **Y**, by doing **Z**.

**Weak**: "Worked on backend services"

**Strong**: "Reduced API response time by 40% (p99: 800ms to 480ms) by implementing Redis caching for the product catalog service, handling 2M+ daily requests"

### Quantify Everything

- Revenue/cost impact: "Saved $50K/month in cloud costs"
- Scale: "Serving 10M+ daily active users"
- Speed: "Reduced deploy time from 45min to 8min"
- Scope: "Led team of 5 engineers across 3 time zones"

### Skills Section

- List languages, frameworks, and tools you can actually discuss in an interview
- Order by proficiency, not alphabetically
- Match keywords from the job description
- Skip soft skills (communication, teamwork) — they don't add signal

### Projects Section (especially for new grads)

- Include 2-3 significant projects with links to live demos or repos
- Describe the **problem solved**, not just the tech used
- Include scale/metrics if available: "1K+ GitHub stars", "500 daily users"

## ATS Optimization

- Use standard job titles: "Software Engineer" not "Code Ninja"
- Include exact keywords from the job description in your bullet points
- Spell out acronyms at least once: "Amazon Web Services (AWS)"
- Save as PDF with the filename format: `FirstName_LastName_Resume.pdf`
- Avoid headers/footers — ATS often can't read them

## Common Mistakes

1. **Listing responsibilities, not accomplishments** — "Responsible for..." vs "Built..."
2. **Including every technology** — focus on what's relevant to the target role
3. **Objective statements** — outdated; use a summary only if you're changing careers
4. **References available upon request** — obvious and wastes a line
5. **Including GPA below 3.5** — it hurts more than it helps
